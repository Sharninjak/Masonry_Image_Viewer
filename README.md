# Masonry Image Viewer

瀑布流本地图片浏览器。无需后端，直接在浏览器中打开 [MasonryViewer.html](MasonryViewer.html) 即可使用。

Source: [wlm3201/Masonry_Image_Viewer](https://github.com/wlm3201/Masonry_Image_Viewer)

## 功能概览

1. 本地图片导入
- 支持目录选择导入（递归扫描）。
- 支持图片文件多选导入。
- 支持拖拽导入与粘贴导入。

2. 图库管理
- 自动把新增来源归档到“当前图库”。
- 支持创建/重命名/编辑描述/删除图库。
- 支持“另存为新图库”。
- 可在欢迎页与图库面板中快速加载已有图库。

3. 布局与浏览
- 支持两种布局：纵向瀑布流（按列）和横向流（按行高）。
- 支持目录树定位跳转到对应目录图片。
- 渐进式加载，降低大目录初始卡顿。

4. 图片查看
- 点击图片进入全屏查看。
- 滚轮缩放、鼠标移动平移。
- 上一张/下一张切换。
- 幻灯片自动播放（3 秒间隔）。

5. 排序与过滤
- 排序：文件名 / 默认 / 日期 / 大小。
- 过滤：黑白漫画、纯色边框、反转。
- 支持宽高比表达式过滤（如 `16:9-`、`-4:3`、`4:3-16:9`）。

## 项目优点

1. 纯前端离线可用
- 不依赖后端服务，适合本地图片快速浏览。

2. 大量图片更流畅
- 扫描阶段优先按扩展名过滤，减少不必要开销。
- 图片按需解码与增量加载，避免一次性占满内存。

3. 长期使用友好
- 图库与来源信息持久化保存。
- 布局、排序、过滤等参数自动记忆。

4. 交互效率高
- 鼠标与键盘双通道操作。
- 目录树跳转、快速翻图、幻灯片支持完整浏览链路。

## 使用环境

- 推荐浏览器：Chrome / Edge（需支持 File System Access API）。
- 建议桌面端系统：Windows / macOS / Linux。
- 安卓端受 API 限制，功能不完整或不可用。

## 快速开始

1. 打开 [MasonryViewer.html](MasonryViewer.html)。
2. 首次进入后，点击页面提示区选择图片文件夹，或使用左上角“+”添加来源。
3. 通过右上角“=”打开参数面板调整布局和过滤。
4. 通过“库”按钮打开图库管理。

## 建议：收藏该文件，便于日常打开

你可以把已打开的 [MasonryViewer.html](MasonryViewer.html) 页面直接加入浏览器书签，后续一键打开。

1. 在浏览器中打开 [MasonryViewer.html](MasonryViewer.html)。
2. 按 `Ctrl + D`（macOS 为 `Cmd + D`）添加书签。
3. 可将书签固定到书签栏，作为“本地图片浏览器”快捷入口。

## 主要操作说明

### 来源导入

- 点击左上角 `+`：
  - `添加文件夹`：导入目录并递归扫描。
  - `添加图片文件`：多选图片文件导入。
- 支持把文件/目录拖到页面中。
- 支持粘贴文件句柄（系统允许时）。

### 图库管理

- 点击 `库` 打开图库面板。
- 常见操作：
  - `加载`：打开该图库。
  - `重命名`：修改图库名称。
  - `编辑描述`：记录备注。
  - `管理来源`：查看并移除来源。
  - `删除`：删除图库。
  - `另存为新图库`：复制当前图库为新条目。

### 布局与数据面板

- `列距/行距/圆角/边框`：外观参数。
- `列数 + 纵向`：瀑布流列数。
- `高度 + 横向`：横向布局目标行高。
- `总数/加载/显示`：统计信息。
- `每次`：每轮加载数量。
- `跳转`：输入序号后回车定位。

## 快捷键与鼠标操作

全屏查看时：

- `←` / `A` / 左键：上一张。
- `→` / `D` / 右键：下一张。
- `Esc` / `S` / 中键：退出全屏。
- `滚轮`：缩放。
- `/`：显示或隐藏帮助面板。

## 数据存储说明

应用会在浏览器本地保存以下数据：

1. 图库元数据
- 存储位置：IndexedDB `masonryViewerDB` 的 `galleries`。
- 内容：图库名称、描述、来源条目、时间戳等。

2. 文件句柄
- 存储位置：IndexedDB `masonryViewerDB` 的 `handles`。
- 说明：浏览器权限策略下，部分环境可能需要重新授权来源。

3. 界面配置
- 存储位置：localStorage。
- 内容：布局类型、列数、间距、排序、过滤状态等。

## 常见问题

1. 为什么加载图库时仍会弹权限请求？
- 浏览器安全策略要求对本地文件/目录访问进行授权或续权。

2. 为什么大图库首次加载较慢？
- 首次需要递归扫描目录并构建索引，后续浏览会明显更流畅。

3. 过滤后图片数量变少是否正常？
- 正常。过滤发生在显示阶段，可先关闭过滤按钮确认原始结果。

4. 为什么推荐 Chrome/Edge？
- 项目依赖 File System Access API，不同浏览器支持程度不同。
